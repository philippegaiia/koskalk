# Documentation

This folder is the project memory for the app.

Use it in two ways:

- `docs/developer/` explains how the application is currently built.
- `docs/specs/` captures the product rules, feature intent, and domain decisions we want to keep true over time.

## Update rule

When a feature changes, update both sides when needed:

- update `docs/developer/` if the implementation or architecture changed
- update `docs/specs/` if the product behavior, domain rules, or UX decision changed
