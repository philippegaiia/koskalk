# Public Dashboard Spec

Last updated: 2026-03-23

## Role

The public dashboard is the user-facing shell around formulation work.

It is not the main product experience by itself. The formulation workbench remains the main vertical surface.

## UI direction

The public dashboard should use custom Tailwind UI, not a generic component library foundation.

The dashboard should feel:

- calm but not soft
- structured
- dense enough for work
- visually aligned with the future formulation page

## Current shell intent

The first dashboard shell should support:

- recipe entry points
- ingredient-related navigation
- compliance entry points
- status and progress context

## Important relationship to formulation

The dashboard exists to get users into the formulation page quickly.

It should not bury recipe work under excessive cards, decorative widgets, or generic app chrome.
